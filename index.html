唯普汽车商户会员体系及积分运营方案
(初稿)

一、方案总则

1. 目的

以真实成交、有效出价、增值服务、合规拉新为核心激励,用清晰等级 + 可感知积分,提升高贡献商户粘性,激活沉睡商户,控制积分成本,推动平台交易与收入增长。

2. 适用对象:全平台买家商户:V0、V1、V2、V3

3. 核心规则

积分:仅靠经营行为获取,无充值、不可转让、不可提现

积分价值:1 积分 = 0.1 元 待定

等级周期:3 个月滚动评估,按近 3 个月成交台数定等级

积分有效期:到账当日起 12 个自然月,到期清零

二、会员等级体系设计

1、等级划分与定级标准:

等级定名:普通会员→金卡会员→白金会员→钻石会员

等级调整:每月 1 号自动更新,按过去 90 天成交计算;达标升级,不达标降级

2、等级有效期与动态调整

  • 有效期:等级当月有效;
  • 等级确定规则:3 个月为一个评估周期,以周期内的交易台次来评级。例如:4.1 日取 1.1-3.31 日的总成交数据(抛仲裁退车)和账号总额(含保证金冻结);3 个月成交台次决定下一个月的等级。每月等级会有变动(原会员等级规则不变)
  • 动态调整:达标自动升级,未达标按规则保级 / 降级。

三、积分获取规则

1、积分说明

  • 积分计算:会员积分基于用户在唯普平台行为的累积分
  • 积分商城的产品设计(一期产品),积分兑换。
  • 积分扣除:详见下方细表

2、积分获取规则(唯一渠道:有效经营行为,无充值入口)

  • 积分作为会员成长核心载体,通过行为赚积分 - 积分促成长 - 成长享权益的逻辑,获取相应的权益奖励
  • 积分不具有货币或现金价值,不可兑换,不可转让
  • 积分可在唯普会员商城兑换各类权益,具体以积分兑换及活动页面展示为准。

3、积分的获取方式

  • 在平台消费可得积分
  • 商户完成打卡任务,获取相应的积分
  • 积分获取无充值渠道,只能通过有效经营行为可获得。

1.1 等级划分

4 级会员:V0 普通 / V1 金卡 / V2 白金 / V3 钻石。评定周期为滚动近 3 个月成交台数,每月 1 号凌晨定时计算,自动升/降级。

等级近 3 月成交账户金额典型权益
V0 普通0 台< 2000积分 1:1 兑换 + 销售顾问
V1 金卡1~3 台≥ 20001:1 兑换 · 历史数据 · 保留价 · 销售顾问 · 1V1 客服
V2 白金4~14 台≥ 200095 折兑换 + V1 全部权益
V3 钻石≥ 15 台≥ 20008 折兑换 + 无理由退车 1 台 + V2 全部权益

1.2 权益落地形式

  • 等级直达权益:到达等级即享,不消耗积分(如历史数据、保留价查询)。
  • 积分兑换优惠券:在商城消耗积分兑换优惠券,按有效期使用(详见 2)。
  • 权益配置唯一数据源:shared/member-config.js → levels[].benefits,前后端同步。

1.3 等级变更规则

  • 评估窗口:T-90d ~ T-1d 的有效成交台数(已付款 + 已过户)。
  • 升降级立即生效,历史优惠券不受等级变化影响。
  • 变更产生审计日志,写入 level_change_log,后台「变更日志」tab 可查。

2.1 积分获取(发放)

积分获取按行为类别配置,数据源 shared/member-config.js → coinEarning[],与 BMS 后台「积分规则」页同源。

类别行为积分说明
  • 积分发放走异步事件,接入事件总线,避免阻塞主业务流程;失败重试 + 死信队列。
  • 节日任务类(如 x2 倍率)需后端支持任务有效期 + 倍率叠加配置。

2.2 积分有效期

{{ coinConfig.validityDesc || '积分按获得月 + 12 个自然月滚动到期' }}。每日凌晨扫描 coin_ledger,将到期积分置为 expired,总余额实时扣减;app/coins.html「即将过期」基于剩余有效期 ≤ 30d。

3.1 业务触发点

  • 下单抵扣:BMS 订单确认页选择一张未使用的优惠券抵扣费用,一单限一张。
  • 售后退车回滚
  • 并发控制:核销接口加行锁 / 版本号,防止同一张卡被多订单重复使用。
