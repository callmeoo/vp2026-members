唯普汽车商户会员体系及积分运营方案
(初稿)

一、方案总则

1. 目的

以真实成交、有效出价、增值服务、合规拉新为核心激励，用清晰等级 + 可感知积分，提升高贡献商户粘性，激活沉睡商户，控制积分成本，推动平台交易与收入增长。

2. 适用对象：全平台买家商户：V0、V1、V2、V3

3. 核心规则

积分：仅靠经营行为获取，无充值、不可转让、不可提现

积分价值：1 积分 = 0.1 元 待定

等级周期：3 个月滚动评估，按近 3 个月成交台数定等级

积分有效期：到账当日起 12 个自然月，到期清零

等级定名：普通会员→金卡会员→白金会员→钻石会员

等级调整：每月 1 号自动更新，按过去 90 天成交计算;达标升级，不达标降级

2、等级有效期与动态调整

  • 有效期：等级当月有效;
  • 等级确定规则：3 个月为一个评估周期，以周期内的交易台次来评级。例如：4.1 日取 1.1-3.31 日的总成交数据(不含退车)和账号总额(含保证金冻结);3 个月成交台次决定下一个月的等级。每月等级会有变动(原会员等级规则不变)
  • 动态调整：达标自动升级，未达标按规则保级 / 降级。

三、积分获取规则

1、积分说明

  • 积分计算：会员积分基于用户在唯普平台行为的累积分
  • 积分商城的产品设计(一期产品)，积分兑换。
  • 积分扣除：详见下方细表

2、积分获取规则(唯一渠道：有效经营行为，无充值入口)

  • 积分作为会员成长核心载体，通过行为赚积分 - 积分促成长 - 成长享权益的逻辑，获取相应的权益奖励
  • 积分不具有货币或现金价值，不可兑换，不可转让
  • 积分可在唯普会员商城兑换各类权益，具体以积分兑换及活动页面展示为准。

3、积分的获取方式

  • 在平台消费可得积分
  • 商户完成打卡任务，获取相应的积分
  • 积分获取无充值渠道，只能通过有效经营行为可获得。

1.1 等级划分

4 级会员：V0 普通 / V1 金卡 / V2 白金 / V3 钻石。评定依据近 3 个自然月成交台数 + 账户，每月 1 日统一定级、当月锁定（口径详见「等级定级逻辑」章）。

成交台次定义变更：成交台次 = 统计周期内已付款的车辆台次，抛仲裁退车车辆。

等级近 3 月成交账户金额典型权益
V0 普通0 台< 2000积分 1:1 兑换 + 销售顾问
V1 金卡1~3 台≥ 20001:1 兑换 · 历史数据 · 保留价 · 销售顾问 · 1V1 客服
V2 白金4~14 台≥ 200095 折兑换 + V1 全部权益
V3 钻石≥ 15 台≥ 20008 折兑换 + 无理由退车 1 台（车辆中标后 24h 内可用）+ V2 全部权益

1.2 权益落地形式

  • 等级直达权益:到达等级即享，不消耗积分(如历史数据、保留价查询)。
  • 积分兑换优惠券:在商城消耗积分兑换优惠券，按有效期使用(详见 2)。
  • 权益配置唯一数据源：shared/member-config.js → levels[].benefits，前后端同步。

1.3 等级变更规则

  • 评估窗口：前 3 个完整自然月(不含当月)的有效成交台数(已付款，不含退车)。
  • 每月 1 日统一定级、当月锁定，月中数据变化下月 1 日生效、不实时升降(详见「等级定级逻辑」章)；历史优惠券不受等级变化影响。
  • 变更产生审计日志，写入 level_change_log，后台「变更日志」tab 可查。

定级口径（月度统一定级）

会员等级每月 1 日统一定级，依据前 3 个完整自然月（不含当月）的累计成交台数 +当前账户（需 ≥ 2000 元）评定。例：6/1 定级看 3~5 月、7/1 看 4~6 月。

当月内成交 / 账户的变化，一律下月 1 日生效。

会员进度条上方文案展示

会员进度条上方的一行文案，按"已生效等级 vs 应得等级"分 6 态展示。涉及页面：移动端 会员中心（app/member.html）、我的（app/profile-loggedin.html）；PC 端 我的首页（pc/member.html）。

什么是「W」：文案的核心数是 W = 前 2 个完整月 + 当月已成交的累计台数（含当月，本月可冲刺）。账户 ≥ 2000 时，就看 W 落在「保级 / 冲下一级 / 已达标」哪一段；账户 < 2000 则资金是硬门槛、先提示。「已生效等级」= 上月 1 日定的、当月锁定，即进度条左侧 hero 大字。

判断逻辑（命中即停）：

① 账户 < 2000 ？
   ├ 已生效 V0      → 当前账户不满 2000 元，充至 2000 元后可参与升级
   └ 已生效 V1~V3   → 当前账户不满 2000 元，下月 1 号有降级风险

② 账户 ≥ 2000 → 看 W（前2月+当月成交台数）落在哪段
   ├ W 低于当前级门槛       → 保级：当月再成交 X 台可保级 V{当前级}，下月 1 号生效
   ├ 够当前级·未到下一级    → 冲下一级：当月再成交 X 台升 V{下一级}，下月 1 号生效
   ├ W 已达下一级门槛       → 已达标：已满足 V× 条件，下月 1 号自动升级
   └ 已 V3·W ≥ 15          → 已封顶：已达最高等级

3.1 业务触发点

  • 下单抵扣:BMS 订单确认页选择一张未使用的优惠券抵扣费用，一单限一张。
  • 售后退车回滚:售后退车执行完毕时，退回原有的成交积分;出价积分不退回。

4.3 资金分配规则

使用无理由退车权益后，原应由买家承担的 2000 元扣罚由平台（唯普）承担，但具体归属各方仍按以下规则拆分（与汇和及业务负责人确认后的版本）：

订单归属资金分配（共 2000 元）
广物汽贸 + 垫款主体 = 汇和 / 门店账号 1200 元 → 汇和 / 门店账号；800 元 → 唯普新收益
广物汽贸 + 其他垫款主体 1200 元 → 唯普云；800 元 → 唯普新收益
非广物汽贸 800 元 → 唯普汽车；1200 元 → 门店

4.4 规则确认过程

① 新规则（与汇和及业务负责人确认）

交易平台侧发起的买家违约仲裁，规则统一调整为「2000 元 → 1200 + 800 拆分」，对应到 3 种订单归属，详见上表。

② 系统原规则（技术提供）

原规则为「2000 元全归一方」，本次调整把广物汽贸两条规则拆出 800 给唯普；非广物汽贸保持不变。

③ 唯普财务回复

财务确认两条调整通过；同时要求前端规则与后端 BMS 规则保持一致。

4.2 发起入口

  • BMS 后台「订单详情」→「申请售后」→ 表单的「款项」与「附件」之间，紧挨「款项」下方展示「买家无理由退车权益」勾选项。
  • 触发条件与商户端一致：类型 = 售后类 且 原因 = 买家违约类 且 处理结果 = 直接退车，三项同时满足才展示。
  • 剩余次数 = 0 时整体不展示。

4.3 BMS 特殊规则

  • 不限 24h 时效：BMS 后台代发起不受订单中标后 24h 限制，只看权益本身的发放/过期日期。与商户端「订单中标后 24h 内可申请」是两件事。
  • 付款状态不影响：未付款 / 已付款均可使用。
  • 驳回归还：售后申请单被驳回时，本次已扣减的权益次数自动回滚 (+1)。
  • 本单使用权益区：订单详情页新增「本单使用的权益」区域，展示已使用的权益清单（保留价查询 / 历史行情查询 / 无理由退车 — 有则增加，无则展示 -）；未使用时显示「本单未使用等级权益」。

4.4 资金分配（与前端一致）

BMS 代为发起售后、已选用无理由退车权益且未自定义款项分配时，资金按以下规则自动处理（与前端规则保持一致）：

订单归属资金分配（共 2000 元）
广物汽贸 + 垫款主体 = 汇和 / 门店账号 1200 元 → 汇和 / 门店账号；800 元 → 唯普新收益
广物汽贸 + 其他垫款主体 1200 元 → 唯普云；800 元 → 唯普新收益
非广物汽贸 800 元 → 唯普汽车；1200 元 → 门店
操作员手动分配优先：若操作员对款项进行了人工分配，按操作员实际执行结果为准，系统不再覆盖。

补偿来源：唯普从自身收益（唯普新收益）中划拨 1200 元给门店 / 汇和或其他利益方。
