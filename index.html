1.1 等级划分

4 级会员:V0 新人 / V1 普通 / V2 银卡 / V3 金卡。评定周期为滚动近 3 个月成交台数,每月 1 号凌晨定时计算,自动升/降级。

等级近 3 月成交账户金额典型权益
V0 新人0 台< 2000积分 1:1 兑换
V1 普通1~2 台≥ 20001:1 兑换 · 历史行情 · 保留价 · 销售顾问 · 1V1 客服
V2 银卡3~9 台≥ 200095 折兑换 + V1 全部权益
V3 金卡≥ 10 台≥ 20008 折兑换 + 专属客服 + 高阶权益

1.2 权益落地形式

  • 等级直达权益:到达等级即享,不消耗积分(如历史行情、保留价查询)。
  • 积分兑换优惠券:在商城消耗积分兑换优惠券,按有效期使用(详见 2)。
  • 权益配置唯一数据源:shared/member-config.js → levels[].benefits,前后端同步。

1.3 等级变更规则

  • 评估窗口:T-90d ~ T-1d 的有效成交台数(已付款 + 已过户)。
  • 升降级立即生效,历史优惠券不受等级变化影响。
  • 变更产生审计日志,写入 level_change_log,后台「变更日志」tab 可查。

2.1 积分获取(发放)

积分获取按行为类别配置,数据源 shared/member-config.js → coinEarning[],与 BMS 后台「积分规则」页同源。单位换算:1 积分 ≈ {{ coinConfig.unitYuan }} 元。

类别行为积分说明
  • 积分发放走异步事件,接入事件总线,避免阻塞主业务流程;失败重试 + 死信队列。
  • 节日任务类(如 x2 倍率)需后端支持任务有效期 + 倍率叠加配置。

2.2 积分有效期

{{ coinConfig.validityDesc || '积分按获得月 + 12 个自然月滚动到期' }}。每日凌晨扫描 coin_ledger,将到期积分置为 expired,总余额实时扣减;app/coins.html「即将过期」基于剩余有效期 ≤ 30d。

2.3 积分消耗(兑换)

商城商品数据源 shared/member-config.js → redeemableRewards[],与 BMS 后台「商城商品配置」同源。

优惠券所需积分适用等级有效期

{{ r.desc }}

  • 兑换价格按等级折扣动态计算:需积分 = 定价 × 等级折扣率(V0/V1 100%、V2 95%、V3 80%)。
  • 流程:扣积分 → 生成 reward_card(状态 unused)→ 推送到「我的优惠券」。
  • 审计:所有兑换写 redeem_log,后台「兑换记录」可查。

2.4 优惠券生命周期

unused(已兑换) → used(已使用,写入业务订单 ID) → expired(过期,定时任务扫描)。

3.1 业务触发点

  • 下单抵扣:BMS 订单确认页选择一张未使用的优惠券抵扣费用,一单限一张。
  • 售后不回滚:退款/售后不退回已使用的优惠券,走补偿策略(退等值积分或替代优惠券,需业务决策)。
  • 并发控制:核销接口加行锁 / 版本号,防止同一张卡被多订单重复使用。

现网已有金币/权益/优惠券等数据,需与创新部/业务方对齐处理口径。本版本重点:

  • 积分配置项:现网金币按 1:1 迁移到新积分(文案全量改为「积分」,变量名保留 coin 不动)。
  • 商城配置项:现有商城商品按新 rewards[] 结构重新录入,旧数据归档不删除。
