调研问卷结果 · 需求点说明 ×
一、问卷列表(默认视图)
  1. 展示所有调研问卷:问卷名称、状态(上线中/已下线/草稿)、发布时间、题目数、提交份数。
  2. 点击行或「查看详情」进入问卷详情页,可查看题目统计与逐份答题记录。
  3. 多份问卷时,「上线中」状态最多 1 份(由 survey-config.js → active:true 控制)。
二、问卷详情视图
  1. 顶部 KPI:累计提交份数 / 来源占比（app / 小程序 / pc）/ 最近提交时间。
  2. 题目维度统计(默认收起):单选/多选题展示选项分布柱状图;简述题不在此卡片显示,具体内容进入答题详情查看。
  3. 提交记录列表:序号 / 用户姓名 / 用户名(系统唯一标识) / 绑定手机号(掩码展示) / 来源 / 提交时间 / 操作(查看答题)。
  4. 「导出全部」将本问卷下所有提交导出为 CSV，列依次为 用户姓名 / 用户名 / 绑定手机号 / 来源 / 提交时间 / Q1~Qn 回答;CSV 内容不脱敏，便于运营线下分析。「清空本问卷」仅清除当前问卷的演示数据,其他问卷不受影响。
三、答题详情(弹窗)
  1. 逐题展示该用户回答:单选含「补充标签」与「补充文本」;多选展示已选项;简述展示原文。
  2. 底部「导出 CSV」可仅导出该单份提交的明细,文件名带用户名 + 时间戳。
  3. 用户信息脱敏:用户名 / 手机号在 CSV 中以掩码展示(如 张** / 138****1234),原始明文仅在系统底层保留。
评审/演示专用按钮 — 不进入生产,无需开发
下列按钮仅用于本 Demo 站点的评审与现场演示,不属于本期需求范围;后端实现时无需任何对应接口或权限。
  1. 预览用户端:新窗口打开 app/survey.html,方便评审人直接体验商户填写流程。
  2. 填充演示数据:本地 localStorage 写入 ~7 条 mock 提交记录(陈伟铭/林晓玥/郑园 等),用于现场演示统计与详情查看。生产由真实提交接口落库,无需此按钮。
  3. 清空本问卷:清除当前问卷在 localStorage 中的 mock 数据,其他问卷不受影响。仅供评审反复演示时复位。
