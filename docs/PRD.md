## 一、方案总则

### 1.1 方案目的

结合唯普汽车商户分层特征与运营需求,搭建**等级清晰、权益梯度明显、积分价值可感知**的商户会员体系。通过积分串联会员成长与平台行为,达成:

- 提升商户平台粘性
- 扩大增值产品(数据查询、复检、过户等)覆盖率
- 提升交易活跃度
- 优化"金字塔型"商户结构
- 推动平台交易规模与多元化收入双增长

### 1.2 核心定位

以 **"交易规模、活跃度、亲密度"** 为会员核心评估维度,打造 **"强转化、建信任、促复购"** 的会员体系,实现:

> **"会员等级越高 → 平台服务越优 → 商户收益越多"** 的正向循环

### 1.3 适用范围

唯普汽车全平台 20 万+ 商户,含:

- V0~V3 各等级商户
- 新入驻商户
- 沉睡商户

## 二、会员等级体系

### 2.1 等级结构

**4 个等级**(纯 4 级,无独立钻石档)

| 代号 | 名称 | 说明 |
|------|------|------|
| V0 | V0 | 新入驻或近 3 个月无成交 |
| V1 | V1 | 入门成交商户,低频使用增值产品 |
| V2 | V2 | 稳定成交商户,中度使用增值产品 |
| V3 | V3 | 高贡献商户,享平台最高权益 |

### 2.2 定级规则

**定级依据**:近 3 个月累计成交台数(滚动计算)

**台数门槛**(经验值,可配置):

| 等级  | 近 3 个月累计成交台数 |
| --- | ------------ |
| V0  | 0 台          |
| V1  | ≥ 3 台        |
| V2  | ≥ 10 台       |
| V3  | ≥ 30 台       |

> 注:阈值在运营后台可配置,无需修改代码。

### 2.3 评估周期

- **滚动计算**:以**每月 1 号凌晨**为结算节点,自动回溯近 3 个月成交台数,重新定级
- 例:4 月 1 日重算时,口径为 **1 月 + 2 月 + 3 月** 的累计成交台数

### 2.4 升降级逻辑

| 场景 | 处理 |
|------|------|
| 成交台数达到更高等级门槛 | 自动升级 |
| 成交台数低于当前等级门槛 | 自动降级 |
| 退车 / 退款 | 对应成交台数扣回,下次重算时可能触发降级 |

**重要**:
- 积分多少**不影响**等级,等级只看成交台数
- **积分不足不会触发等级下调**
- 退车/退款只影响成交台数,不扣积分

## 三、积分体系

### 3.1 基本说明

- 统一名称:**积分**(文档内部与"积分"同义)
- 基于商户在唯普平台的**有效经营行为**累积
- **不具有货币或现金价值**,不可兑换现金、不可转让
- 可在唯普 APP / 小程序的**会员商城**兑换权益卡

### 3.2 积分获取规则

| 行为 | 积分数 | 去重维度 | 前提条件 |
|------|--------|----------|----------|
| **出价** | +1 枚 | 用户 ID × 车辆 ID | 登录状态 |
| **转发朋友圈** | +2 枚 | 用户 ID × 车辆 ID | 登录状态 |
| **成交** | +50 枚 | 每台次 | 过户完成 / 付款完成后发放 |

**细则说明**:

**① 出价积分**
- 用户登录后,对某台车出价成功即发放 1 枚
- 同一用户对同一台车,**无论出价几次都只发 1 枚**
- 去重键:`user_id + car_id`

**② 转发积分**
- 用户把车辆详情页**转发到微信朋友圈**即发放 2 枚
- 同一用户转发同一台车,**只发 1 次**(2 枚)
- 去重键:`user_id + car_id`
- **必须登录**才计入,未登录状态转发不发放
- 受**每日上限**与**总量上限**约束(具体数字由运营配置)

**③ 成交积分**
- 成交后的**过户完成 或 付款完成**时才发放
- 每台固定 50 枚
- 因为在过户/付款后才发放,所以**无"退款扣回"场景**

### 3.3 积分有效期

- 自获得当月起 **12 个自然月内** 有效
- 到期自动清零
- 不同批次积分分别计算过期时间(FIFO)

### 3.4 积分扣减规则

- **不因退车/退款扣减**(原因:积分是在过户/付款完成后才发放)
- **消耗**:用户兑换权益卡时按标价扣减
- **过期**:到期未使用自动清零

### 3.5 使用场景

- 一期(当前):**仅限唯车拍小程序**内使用
- 唯一用途:在会员商城兑换权益卡
- 后续可扩展其他消耗场景

## 四、权益体系

### 4.1 体系定位

权益是积分的核心消耗出口,同时部分高价值权益作为高等级会员的**等级直达**(无需积分兑换)。

权益分两类:
- **积分兑换型**:用户用积分在商城兑换,有库存与有效期
- **等级直达型**:商户达到对应等级后**自动享有**,无需兑换

### 4.2 各等级权益总览

| 等级     | 折扣                | 数据查询(可兑换)          | 服务类(等级直达)                 | 退车        |
| ------ | ----------------- | ------------------ | ------------------------- | --------- |
| **V0** | 积分兑换无折扣          | 基础查询卡              | —                         | —         |
| **V1** | 积分兑换无折扣          | 全量查询卡 + 历史数据 + 保留价 | 1V1 客服、专属销售顾问             | —         |
| **V2** | **过户 / 复检 9.5 折** | 同 V1               | 同 V1                      | —         |
| **V3** | **过户 / 复检 8 折**   | 同 V1               | V1 全部 + 代送车上门(代物流)、免费停车服务 | 无理由退车 1 台 |

> 说明:
> - V1 的数据查询权益**通过积分兑换获得**,不是免费直享
> - 服务类(1V1 客服、专属销售顾问、代送车上门、免费停车)是**等级直达**,达到等级后自动开通,无需积分
> - **代签名**权益**待定**(本期不展示)
> - "无理由退车"和"无偿退车"是同一概念,文档统一使用 **"无理由退车"**

### 4.4 权益卡使用规则

| 项目 | 规则 |
|------|------|
| 兑换路径 | 唯普 APP / 小程序 → 我的 → 权益卡 / 会员商城 |
| 库存控制 | 先兑先得,受库存限制 |
| 等级限制 | 高等级权益卡仅对应等级可兑换;降级后**未使用的权益卡自动失效** |
| 有效期 | 兑换之日起 7~30 天(以卡面规则为准) |
| 转让 | **不可**转让、折现、退换 |
| 退车退款 | 对应权益**同步作废**(权益本身,不影响积分) |

## 五、运营与后台能力

### 5.1 需要后台可配置的规则

| 模块 | 配置项 |
|------|--------|
| 等级规则 | 各级台数门槛、评估周期、降级策略 |
| 积分规则 | 出价/转发/成交每项的积分数、转发每日上限、转发总量上限 |
| 积分有效期 | 默认 12 个月,可调整 |
| 权益卡目录 | 新增/下架权益卡、积分价、等级门槛、库存、有效期 |
| 等级直达权益 | 配置每个等级自动享有的服务类权益 |
| 活动推送 | 规则变更通知(APP 端 / 短信 / 弹窗) |

### 5.2 需要后台可查询的数据

| 模块 | 查询项 |
|------|--------|
| 商户档案 | 等级、当前积分余额、各渠道累计积分、即将过期积分 |
| 积分流水 | 按用户/时间/行为类型筛选 |
| 兑换记录 | 按商户/权益卡/时间筛选,支持退回库存 |
| 等级变更历史 | 每月重算后的变更记录 |
| 数据看板 | 各等级商户分布、积分发放/消耗趋势、权益卡热度 |

## 六、技术与合规说明

### 6.1 规则变更通知

- 平台有权调整会员规则、积分规则、权益卡目录
- 变更前通过 **APP 弹窗 / 站内信 / 短信** 至少一种方式通知
- 更新内容在指定生效日期后执行

### 6.2 反作弊

- 出价/转发的**去重维度**本身就是基础防刷
- 系统应记录完整日志以备追溯
- 异常刷分行为由运营后台的风控模块处理(本期不展开)

### 6.3 数据隐私

- 商户积分流水、兑换记录仅本人可查看
- 运营后台仅限授权人员访问
