唯普汽车 · 商户会员体系 PRD(技术评审稿)

版本 v1.1 更新 2026-04-24 文档用途:研发技术评审

基于已生成的原型页(app/ + admin/)与 shared/member-config.js 配置,梳理会员、积分、权益三大体系的规则与实现要点,用于研发排期与技术评审。

一、会员等级和权益

1.1 等级划分

4 级会员:V0 新人 / V1 普通 / V2 银卡 / V3 金卡。评定周期为滚动近 3 个月成交台数,每月 1 号凌晨定时计算,自动升/降级。

等级近 3 月成交账户金额典型权益
V0 新人0 台< 2000积分 1:1 兑换
V1 普通1~2 台≥ 20001:1 兑换 · 历史行情 · 保留价 · 销售顾问 · 1V1 客服
V2 银卡3~9 台≥ 200095 折兑换 + V1 全部权益
V3 金卡≥ 10 台≥ 20008 折兑换 + 专属客服 + 高阶权益

1.2 权益落地形式

  • 等级直达权益:到达等级即享,不消耗积分(如历史行情、保留价查询)。
  • 积分兑换权益卡:在商城消耗积分兑换权益卡,按有效期使用(详见第二节)。
  • 权益配置唯一数据源:shared/member-config.js → levels[].benefits,前后端同步。

1.3 等级变更规则

  • 评估窗口:T-90d ~ T-1d 的有效成交台数(已付款 + 已过户)。
  • 升降级立即生效,历史权益卡不受等级变化影响。
  • 变更产生审计日志,写入 level_change_log 表,后台"变更日志" tab 可查。

二、积分商城

2.1 积分获取(发放)

积分获取按行为类别配置,全部规则在 member-config.coinEarning[]。研发关注点:

  • 交易类:成交 1 台 +50 积分;出价 1 次 +10 积分,不封顶。
  • 认证类:首次实名 +1~5、首次充值 +1~5,仅一次性。
  • 邀请类:邀请好友注册 +2,推荐首笔成交 +2(30 天内成交日核算,双方都得)。
  • 增值服务类:物流/代驾/复检/代办等,每单 +5 积分。
  • 节日任务类:活动期内出价积分 x2,后端需支持任务有效期 + 倍率配置。
  • 积分发放走异步事件,接入事件总线,避免阻塞主业务流程;失败重试 + 死信队列。

2.2 积分有效期

积分按获得月 + 12 个自然月滚动到期(非自然年末清零)。每日凌晨扫描 coin_ledger,将到期积分状态置为 expired,总余额实时扣减;app/coins.html 的"即将过期"提醒基于剩余有效期 ≤ 30d。

2.3 积分消耗(兑换)

  • 商城商品见 member-config.rewards[],单项字段:coin(所需积分)、scope(适用等级)、validityDays(权益卡有效期)。
  • 兑换价格按等级折扣动态计算:需积分 = 定价 × 等级折扣率(V0/V1 100%、V2 95%、V3 80%)。
  • 兑换流程:扣积分 → 生成 reward_card 记录(状态 unused)→ 推送到"我的权益卡"。
  • 兑换审计:所有兑换写 redeem_log,后台"兑换记录"可查。

2.4 权益卡生命周期

unused(已兑换) → used(已使用,写入业务订单 ID) → expired(过期,定时任务扫描)。不支持退款/回滚。

三、客户端消耗逻辑

3.1 业务触发点

  • 下单抵扣:在 BMS 订单确认页选择一张未使用的权益卡抵扣费用,一单限一张。
  • 售后不回滚:订单退款/售后不退回已使用的权益卡,走补偿策略(退等值积分或替代权益卡,需业务决策)。
  • 并发控制:核销接口加行锁 / 版本号,防止同一张卡被多订单重复使用。

3.2 接口清单(详见 docs/API.md)

接口说明
GET /api/member/profile返回当前等级 + 积分余额 + 权益列表
GET /api/coin/ledger积分明细(支持筛选 + 分页)
POST /api/reward/redeem兑换权益卡(幂等键 = 用户 + 商品 + 时间戳)
POST /api/reward/use下单时核销一张权益卡,返回抵扣金额

四、历史数据处理(待定)

现网已有金币/权益/优惠券等数据,需与创新部/业务方对齐处理口径。本版本重点:

  • 积分配置项:现网金币按 1:1 迁移到新积分(文案全量改为"积分",变量名保留 coin 不动)。
  • 商城配置项:现有商城商品按新 rewards[] 结构重新录入,旧数据归档不删除。
